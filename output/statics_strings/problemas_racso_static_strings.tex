\section{Herramientas comunes}

\subsection{Sparse table para RMQ}
La sparse table guarda respuestas de rangos de longitud potencia de dos:
\[
st[k][i]=\min(A[i],\ldots,A[i+2^k-1]).
\]
La transicion es:
\[
st[k][i]=\min(st[k-1][i],st[k-1][i+2^{k-1}]).
\]
Para consultar un rango de longitud $len$, se toma:
\[
k=\lfloor\log_2 len\rfloor.
\]
Esto significa que $2^k$ es la potencia de dos mas grande que cabe dentro del
rango. Se usan dos bloques de longitud $2^k$: uno pegado al inicio y otro al
final. En minimo pueden solaparse porque $\min(x,x)=x$.

\subsection{Suffix array, rank y LCP}
Un sufijo de $S$ es una parte que empieza en alguna posicion y llega hasta el
final. Si $S=\texttt{banana}$:
\[
\text{sufijos: banana, anana, nana, ana, na, a.}
\]
El suffix array $\SA$ guarda las posiciones de los sufijos en orden
lexicografico. El arreglo inverso $\rankv$ cumple:
\[
\rankv[i]=\text{posicion del sufijo }S[i..]\text{ dentro de }\SA.
\]
El LCP array guarda cuanto comparten sufijos vecinos:
\[
\LCP[i]=\lcp(S[\SA[i-1]..],S[\SA[i]..]).
\]
Para dos sufijos arbitrarios $i,j$, si sus ranks son $a<b$:
\[
\lcp(i,j)=\min(\LCP[a+1],\ldots,\LCP[b]).
\]
Por eso se preprocesa RMQ sobre $\LCP$.

\subsection{LCA y caminos en arbol}
En un arbol fijo, $\lca(u,v)$ parte el camino $u-v$ en dos caminos hacia arriba.
Binary lifting guarda:
\[
up[v][j]=\text{ancestro }2^j\text{ de }v.
\]
Si ademas se guarda el maximo de aristas en cada salto, se pueden responder
queries de ``max edge on path''.

\section{Static RMQ}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/694840/problem/A}

\subsection{Enunciado oficial resumido}
Se da una secuencia de $N$ enteros y $Q$ consultas. Cada consulta contiene
$l,r$ con $0\le l<r\le N$ y pide imprimir:
\[
\min(a_l,a_{l+1},\ldots,a_{r-1}).
\]
El rango es \textbf{semiabierto}: incluye $l$ y excluye $r$. Restricciones
relevantes: $N,Q\le 5\cdot 10^5$.

\subsection{Como reconocerlo}
El arreglo no cambia y hay muchas consultas de minimo. Eso grita sparse table:
preprocesamiento $\bigO(N\log N)$ y query $\bigO(1)$.

\subsection{Solucion de examen}
Construyo $st[k][i]$, el minimo del bloque que empieza en $i$ y tiene longitud
$2^k$. Para responder $[l,r)$, su longitud es $len=r-l$. Tomo
$k=\lfloor\log_2 len\rfloor$. Los bloques:
\[
[l,l+2^k)\quad\text{y}\quad [r-2^k,r)
\]
cubren todo $[l,r)$. La respuesta es:
\[
\min(st[k][l],st[k][r-2^k]).
\]

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
build(A):
  lg[1] = 0
  for i = 2..N: lg[i] = lg[i/2] + 1
  for i = 0..N-1: st[0][i] = A[i]
  for k = 1..lg[N]:
    for i = 0; i + (1<<k) <= N; i++:
      st[k][i] = min(st[k-1][i],
                     st[k-1][i + (1<<(k-1))])

query(l, r): // rango [l, r)
  len = r - l
  k = lg[len]
  return min(st[k][l], st[k][r - (1<<k)])
\end{lstlisting}

\subsection{Correctitud}
Por induccion, $st[k][i]$ guarda el minimo de su bloque: para $k=0$ es un solo
elemento; para $k>0$ el bloque se parte en dos mitades ya calculadas. En una
consulta, los dos bloques de longitud $2^k$ cubren todo el rango semiabierto
$[l,r)$. El minimo global esta en al menos uno de ellos. Si se solapan, el
minimo no cambia porque la operacion es idempotente. Por tanto la respuesta es
correcta.

\subsection{Complejidad}
Preprocesamiento $\bigO(N\log N)$, memoria $\bigO(N\log N)$, query $\bigO(1)$.
Detalle de examen: aqui $len=r-l$, no $r-l+1$, porque el statement usa rango
semiabierto $[l,r)$.

\section{Suffix array y LCP array}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/695758/problem/A}

\subsection{Enunciado oficial resumido}
Dada una cadena $s$ de longitud hasta $5\cdot 10^5$, imprimir dos lineas:
el suffix array indexado desde $0$ y el LCP array correspondiente. La cadena
contiene letras minusculas.

\subsection{Que debes explicar en examen}
Debes definir sufijo, suffix array, rank y LCP. Luego puedes proponer
Manber-Myers para construir $\SA$ y Kasai para construir $\LCP$.

\subsection{Manber-Myers desde cero}
Manber-Myers no ordena sufijos completos desde el inicio. Ordena prefijos de
longitud:
\[
1,2,4,8,\ldots
\]
El arreglo $rank[i]$ significa: el orden/clase del prefijo de longitud actual
que empieza en $i$. Si ya se ordenar prefijos de longitud $len$, entonces el
prefijo de longitud $2len$ que empieza en $i$ se representa por:
\[
(rank[i],rank[i+len]).
\]
Ordenar esos pares ordena por longitud doble.

\subsection{Kasai}
Kasai construye el LCP array despues de tener $\SA$. Usa el rank inverso:
\[
rank[i]=\text{posicion del sufijo }i\text{ en }\SA.
\]
Para cada sufijo $i$, mira su vecino anterior en $\SA$:
\[
j=\SA[rank[i]-1].
\]
Calcula $\lcp(S[i..],S[j..])$. La optimizacion es que si el LCP anterior era
$h$, al pasar de $i$ a $i+1$ el LCP baja como mucho en $1$, asi que se empieza
desde $h-1$.

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
build_sa(S):
  sa = [0,1,...,n-1]
  rank[i] = S[i]
  for len = 1; len < n; len *= 2:
    sort sa by (rank[i], rank[i+len] or -1)
    newRank[sa[0]] = 0
    for p = 1..n-1:
      newRank[sa[p]] = newRank[sa[p-1]]
      if pair(sa[p]) != pair(sa[p-1]):
        newRank[sa[p]]++
    rank = newRank
  return sa

kasai(S, sa):
  for p = 0..n-1: rank[sa[p]] = p
  h = 0
  for i = 0..n-1:
    if rank[i] == 0: continue
    j = sa[rank[i]-1]
    while i+h<n && j+h<n && S[i+h]==S[j+h]: h++
    lcp[rank[i]] = h
    if h > 0: h--
\end{lstlisting}

\subsection{Correctitud}
Manber-Myers es correcto por induccion: si $rank$ ordena prefijos de longitud
$len$, entonces el par de ranks ordena las dos mitades de un prefijo de longitud
$2len$. Al duplicar hasta cubrir toda la cadena, los prefijos son los sufijos
completos. Kasai es correcto porque calcula el LCP de cada sufijo con su vecino
lexicografico anterior, y la reutilizacion de $h$ no omite caracteres: al
eliminar el primer caracter de dos sufijos, su prefijo comun disminuye como
mucho en uno.

\subsection{Complejidad}
Manber-Myers cuesta $\bigO(n\log n)$ si cada ronda ordena en $\bigO(n)$ con
counting/radix sort; con sort comparativo puede ser $\bigO(n\log^2 n)$. Kasai
cuesta $\bigO(n)$.

\section{Busqueda de subcadenas}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/695758/problem/B}

\subsection{Enunciado oficial resumido}
Se da un texto fijo $t$ y $n$ consultas online. Cada consulta trae una cadena
$s_i$ y se debe responder si $s_i$ aparece como subcadena de $t$. La suma total
de longitudes de patrones es hasta $3\cdot 10^5$. El statement pide hacer flush
despues de cada respuesta por el formato online.

\subsection{Solucion de examen}
Construyo el suffix array de $t$. Una aparicion de $P$ en $t$ equivale a un
sufijo de $t$ que empieza con $P$. En el suffix array, todos los sufijos que
tienen prefijo $P$ forman un intervalo contiguo. Hago dos busquedas binarias
para encontrar ese intervalo. Si es no vacio, respondo Yes; si es vacio, No.

\subsection{Por que el intervalo es contiguo}
En orden lexicografico, todos los strings que comparten un prefijo fijo $P$
quedan juntos. Si hubiera un sufijo sin prefijo $P$ entre dos que si lo tienen,
en el primer caracter donde difiere tendria que ser simultaneamente mayor que
uno y menor que otro con el mismo prefijo, contradiciendo el orden.

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
has_prefix(pos, P):
  // true si P es prefijo de t[pos..]
  for i = 0..|P|-1:
    if pos+i == n || t[pos+i] != P[i]: return false
  return true

compare_suffix_with_pattern(pos, P):
  // -1 si sufijo < P, 0 si P es prefijo, 1 si sufijo > P
  for i = 0..|P|-1:
    if pos+i == n: return -1
    if t[pos+i] < P[i]: return -1
    if t[pos+i] > P[i]: return 1
  return 0

exists(P):
  L = first position in SA with suffix >= P
  return L < n && has_prefix(SA[L], P)
\end{lstlisting}

\subsection{Correctitud}
Cada aparicion de $P$ empieza en alguna posicion $i$, y entonces el sufijo
$t[i..]$ tiene prefijo $P$. A la inversa, si un sufijo tiene prefijo $P$, hay
una aparicion en su posicion inicial. La busqueda binaria encuentra el primer
sufijo no menor que $P$; si ese sufijo tiene prefijo $P$, existe una aparicion,
y si no, ningun sufijo posterior puede iniciar el bloque correcto. Por la
contiguidad del bloque, la respuesta es correcta.

\subsection{Complejidad}
Preprocesamiento $\bigO(|t|\log |t|)$. Cada consulta cuesta
$\bigO(|P|\log |t|)$ con comparacion directa. El flush no cambia complejidad,
solo es requisito de interaccion.

\section{Contando subcadenas}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/695758/problem/C}

\subsection{Enunciado oficial resumido}
Igual que el problema anterior, se da un texto fijo $t$ y consultas online con
patrones $s_i$. Esta vez se debe imprimir cuantas veces aparece $s_i$ como
subcadena de $t$.

\subsection{Solucion de examen}
Construyo suffix array. Para cada patron $P$, encuentro con dos busquedas
binarias el intervalo $[L,R)$ de sufijos que empiezan con $P$. El numero de
apariciones es:
\[
R-L.
\]

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
count_occurrences(P):
  L = lower_bound_first_suffix_geq(P)
  R = lower_bound_first_suffix_strictly_after_prefix(P)
  return R - L
\end{lstlisting}

\subsection{Correctitud}
La correspondencia entre ocurrencias y sufijos con prefijo $P$ es biyectiva:
cada ocurrencia define un sufijo y cada sufijo con prefijo $P$ define una
ocurrencia. Como esos sufijos forman un bloque contiguo en $\SA$, el tamano del
bloque es exactamente el numero de ocurrencias.

\subsection{Complejidad}
$\bigO(|t|\log |t|)$ para construir. Por consulta,
$\bigO(|P|\log |t|)$ en la version simple. Puede optimizarse con LCP, pero para
respuesta teorica del curso basta el intervalo de suffix array.

\section{Subcadenas diferentes}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/695758/problem/D}

\subsection{Enunciado oficial resumido}
Dada una cadena $s$ de longitud hasta $5\cdot 10^5$, contar cuantas subcadenas
distintas tiene.

\subsection{Solucion de examen}
Toda subcadena es prefijo de algun sufijo. Si procesamos sufijos en orden de
$\SA$, el sufijo $\SA[i]$ tiene $n-\SA[i]$ prefijos, pero los primeros
$\LCP[i]$ ya aparecieron como prefijos del sufijo anterior. Entonces:
\[
ans=\sum_i \big((n-\SA[i])-\LCP[i]\big)
    =\frac{n(n+1)}2-\sum_i\LCP[i].
\]

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
distinct_substrings(s):
  sa = build_sa(s)
  lcp = kasai(s, sa)
  ans = 1LL*n*(n+1)/2
  for i = 1..n-1:
    ans -= lcp[i]
  return ans
\end{lstlisting}

\subsection{Correctitud}
Cada subcadena aparece como prefijo de al menos un sufijo. En orden
lexicografico, cuando llegamos a un sufijo, los prefijos que ya fueron vistos
son exactamente los que comparte con el sufijo anterior, medidos por $\LCP[i]$.
Los prefijos mas largos son nuevos. Por tanto sumar aportes nuevos cuenta cada
subcadena distinta una sola vez.

\subsection{Complejidad}
Construccion de $\SA+\LCP$: $\bigO(n\log n)$ con Manber-Myers y Kasai. Conteo:
$\bigO(n)$.

\section{Subcadena en comun mas larga}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/695758/problem/E}

\subsection{Enunciado oficial resumido}
Dadas dos cadenas $s,t$ de longitud hasta $10^5$, imprimir una subcadena comun
de longitud maxima. Si hay varias, se debe imprimir la lexicograficamente minima.

\subsection{Solucion de examen}
Concateno:
\[
T=s\#t\$,
\]
con separadores que no aparecen en las cadenas. Construyo $\SA$ y $\LCP$ de
$T$. Recorro sufijos vecinos. Si uno viene de $s$ y otro de $t$, su LCP es una
subcadena comun. Tomo la mayor longitud; en empate, la lexicograficamente menor.

\subsection{Detalle del empate lexicografico}
Como el suffix array esta en orden lexicografico, las substrings candidatas se
pueden comparar por su posicion/rank. Si dos candidatas tienen la misma
longitud, se conserva la que aparece antes lexicograficamente. En una
implementacion simple se puede comparar las substrings de longitud $best$.

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
LCS(s, t):
  T = s + "#" + t + "$"
  sa = build_sa(T)
  lcp = kasai(T, sa)
  bestLen = 0; bestPos = -1
  for i = 1..|T|-1:
    if origin(sa[i]) != origin(sa[i-1]):
      candLen = lcp[i]
      candPos = sa[i]
      if candLen > bestLen:
        update
      else if candLen == bestLen:
        choose lexicographically smaller substring
  return T.substr(bestPos, bestLen)
\end{lstlisting}

\subsection{Correctitud}
Sea $X$ una subcadena comun optima. Hay un sufijo de $s$ y uno de $t$ que
tienen prefijo $X$. Todos los sufijos con prefijo $X$ forman un bloque en
$\SA$. Dentro de ese bloque existe un par adyacente de origen distinto; el LCP
de ese par es al menos $|X|$. Por tanto, revisar vecinos de origen distinto
encuentra una subcadena comun de longitud maxima. El criterio de desempate
elige la menor en orden lexicografico.

\subsection{Complejidad}
$\bigO((|s|+|t|)\log(|s|+|t|))$ con Manber-Myers. Recorrido lineal.

\section{Ordenando subcadenas}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/695758/problem/F}

\subsection{Enunciado oficial resumido}
Se da una cadena $s$ de longitud hasta $4\cdot 10^5$ con caracteres ASCII
imprimibles. Luego se dan $n$ subcadenas por pares $(l,r)$, indexados desde $1$.
Se deben imprimir esos pares ordenados por el valor lexicografico de la
subcadena. Si dos subcadenas son iguales como palabra, se desempata por el par:
primero menor $l$, luego menor $r$.

\subsection{Solucion de examen}
Para comparar $s[l_1..r_1]$ y $s[l_2..r_2]$, calculo:
\[
x=\lcp(l_1,l_2)
\]
con $\SA+\LCP+\RMQ$, recortado por la longitud minima. Si $x$ cubre la mas
corta, gana la mas corta; si tienen igual longitud, desempato por $(l,r)$. Si
no, comparo el primer caracter distinto.

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
lcp_suffix(i, j):
  if i == j: return n-i
  a = rank[i]; b = rank[j]
  if a > b: swap(a,b)
  return rmq_lcp(a+1, b)

less_substring(A=(l1,r1), B=(l2,r2)):
  len1 = r1-l1+1; len2 = r2-l2+1
  x = min(lcp_suffix(l1,l2), min(len1,len2))
  if x < min(len1,len2):
    return s[l1+x] < s[l2+x]
  if len1 != len2: return len1 < len2
  return A < B  // desempate por l, luego r
\end{lstlisting}

\subsection{Correctitud}
El primer caracter donde difieren dos strings determina el orden lexicografico.
El LCP de los sufijos desde $l_1$ y $l_2$ da exactamente cuantos caracteres
iniciales coinciden. Si la coincidencia termina antes que ambas subcadenas, el
siguiente caracter decide. Si una subcadena se acaba, entonces es prefijo de la
otra y la mas corta es menor. Si son iguales, el statement pide desempatar por
el par.

\subsection{Complejidad}
Preprocesamiento $\bigO(n\log n)$. Cada comparacion $\bigO(1)$. Ordenar $q$
subcadenas cuesta $\bigO(q\log q)$.

\section{Bordes}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/695758/problem/G}

\subsection{Enunciado oficial resumido}
Una cadena $t$ es borde de $s$ si es a la vez prefijo y sufijo de $s$. El
statement incluye la cadena vacia como borde. Dada una cadena $s$ de longitud
hasta $4\cdot 10^5$, se pide el numero total de bordes de \textbf{todas las
subcadenas} de $s$.

\subsection{Correccion importante}
Este problema no pide solamente los bordes de la cadena completa. Pide sumar,
para cada subcadena de $s$, cuantos bordes tiene. Por eso una solucion simple
con KMP sobre la cadena completa no alcanza.

\subsection{Idea con suffix array}
Cada subcadena tiene siempre el borde vacio. Eso aporta:
\[
\frac{n(n+1)}2.
\]
Ahora contamos bordes no vacios. Un borde no vacio de longitud $b$ en una
subcadena aparece cuando hay dos substrings iguales de longitud $b$: una al
inicio y otra al final de esa subcadena. Por cada par de sufijos $(i,j)$, cada
longitud $b\le \lcp(i,j)$ produce un borde no vacio. Entonces:
\[
ans=\frac{n(n+1)}2+\sum_{i<j}\lcp(i,j).
\]

\subsection{Como calcular la suma de LCP de todos los pares}
Si dos sufijos tienen ranks $a<b$, su LCP es el minimo en el intervalo
$\LCP[a+1..b]$. Por tanto:
\[
\sum_{i<j}\lcp(i,j)
\]
es la suma de minimos de todos los subarreglos del arreglo LCP. Se calcula con
pila monotona en $\bigO(n)$.

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
sum_pair_lcp(lcp[1..n-1]):
  stack = empty // pares (height, count)
  cur = 0
  total = 0
  for h in lcp[1..n-1]:
    cnt = 1
    while stack not empty && stack.top.height >= h:
      cur -= stack.top.height * stack.top.count
      cnt += stack.top.count
      pop
    push (h, cnt)
    cur += h * cnt
    total += cur
  return total

answer = n*(n+1)/2 + sum_pair_lcp(lcp)
\end{lstlisting}

\subsection{Correctitud}
Para cada par de posiciones $i<j$, si los sufijos comparten $L$ caracteres,
entonces para cada $b=1..L$ las substrings de longitud $b$ que empiezan en $i$
y $j$ son iguales. Esas dos copias forman el prefijo y sufijo de la subcadena
$s[i..j+b-1]$, asi que aportan un borde no vacio. A la inversa, cualquier borde
no vacio de cualquier subcadena define exactamente un par de substrings iguales
en sus extremos, y por tanto un par de sufijos con LCP suficiente. La pila
monotona suma el minimo de todos los intervalos de LCP, que equivale a la suma
de LCP de todos los pares de sufijos.

\subsection{Complejidad}
Construir $\SA+\LCP$: $\bigO(n\log n)$ o $\bigO(n)$ segun algoritmo. Sumar con
pila: $\bigO(n)$. Usar \texttt{long long}.

\section{Refran}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/695758/problem/H}

\subsection{Enunciado oficial resumido}
Se da un arreglo de $n$ enteros entre $1$ y $m$, con $m\le 10$. Una subsecuencia
consecutiva del arreglo se llama refran. Hay que encontrar un refran que
maximice:
\[
\text{longitud}\times\text{numero de apariciones}.
\]
Se imprime ese valor maximo, la longitud y el refran.

\subsection{Como verlo}
Aunque el input es un arreglo de enteros, se trata igual que un string sobre un
alfabeto pequeno. Las ``subcadenas'' son subarreglos consecutivos. Se construye
suffix array del arreglo.

\subsection{Solucion}
Una subcadena que aparece $f$ veces corresponde a $f$ sufijos consecutivos que
comparten un prefijo de cierta longitud. Si un intervalo de LCP tiene minimo
$h$ y cubre $w$ barras, entonces hay $w+1$ sufijos que comparten un prefijo de
longitud $h$. El valor candidato es:
\[
h\cdot (w+1).
\]
Esto es el mismo patron que ``largest rectangle in histogram'', usando LCP como
histograma.

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
best_refrain():
  sa = suffix_array_over_integer_array(a)
  lcp = kasai_over_array(a, sa)
  bestValue = n; bestLen = n; bestPos = 0 // frecuencia 1
  use monotonic stack over lcp:
    when height h is popped over width w:
      freq = w + 1
      value = h * freq
      if value > bestValue:
        bestValue = value
        bestLen = h
        bestPos = sa[some suffix in interval]
  print a[bestPos .. bestPos+bestLen-1]
\end{lstlisting}

\subsection{Correctitud}
Si un subarreglo $X$ aparece $f$ veces, los sufijos que empiezan en esas
apariciones comparten $X$ como prefijo y forman un bloque en el suffix array.
La longitud comun maxima del bloque es el minimo LCP interno. La pila monotona
considera cada posible minimo de un bloque maximal, asi que considera el bloque
que representa cualquier refran optimo. Por tanto encuentra el maximo producto.

\subsection{Complejidad}
$\bigO(n\log n)$ con construccion simple de suffix array, $\bigO(n)$ para LCP y
pila. El alfabeto pequeno ayuda, pero la idea teorica es suffix array + LCP
histogram.

\section{Substring periodica}
\textbf{Link oficial.} \url{https://codeforces.com/group/AmklRC8lhZ/contest/695758/problem/I}

\subsection{Enunciado oficial resumido}
Dada una cadena $s$ de longitud hasta $4\cdot 10^5$, encontrar el mayor entero
$k$ tal que exista una subcadena no vacia que pueda escribirse como la
concatenacion de $k$ cadenas iguales.

\subsection{Que significa}
Buscamos el maximo exponente de una repeticion. Por ejemplo, si aparece
\[
d^k = d+d+\cdots+d,
\]
entonces esa subcadena tiene periodo $|d|$ y longitud $k|d|$.

\subsection{Teoria necesaria}
Para probar si una zona tiene periodo $p$, necesitamos comparar substrings
desplazadas:
\[
s[l..r-p]=s[l+p..r].
\]
Con $\SA+\LCP+\RMQ$, esta igualdad se verifica en $\bigO(1)$. Para encontrar
la respuesta maxima de forma eficiente, una solucion completa enumera
\textbf{runs} o repeticiones maximales. Un run es un intervalo maximal con un
periodo minimo $p$ y longitud al menos $2p$.

\subsection{Solucion teorica aceptable}
Preprocesa LCP hacia la derecha y LCP en la cadena invertida para obtener LCS
(longest common suffix). Para un posible periodo $p$ y una frontera entre dos
bloques iguales, calculas:
\[
R=\lcp(i,i+p)
\]
extension hacia la derecha, y:
\[
L=\text{LCS}(i-1,i+p-1)
\]
extension hacia la izquierda. Entonces el bloque maximal con periodo $p$
alrededor de esa frontera tiene longitud aproximadamente:
\[
L+p+R.
\]
El numero de repeticiones candidatas es:
\[
\left\lfloor\frac{L+p+R}{p}\right\rfloor.
\]
Una implementacion completa debe enumerar fronteras representativas sin probar
todos los pares; esto se hace con algoritmos de runs o con tecnicas de
divide-and-conquer. La idea de examen es justificar la verificacion de periodos
con LCP/LCS.

\subsection{Pseudocodigo conceptual}
\begin{lstlisting}[language=C++]
is_period(l, len, p):
  return lcp_suffix(l, l+p) >= len-p

max_power_by_runs():
  build LCP structure on S and reversed(S)
  ans = 1
  for each maximal run (L, R, p):
    ans = max(ans, (R-L+1) / p)
  return ans
\end{lstlisting}

\subsection{Correctitud}
Una subcadena es $d^k$ si tiene periodo $p=|d|$ y longitud al menos $kp$. La
igualdad desplazada por $p$ caracteriza exactamente a las cadenas con periodo
$p$. Los runs son intervalos maximales donde esa propiedad se mantiene; cualquier
repeticion esta contenida en algun run de su periodo. Por eso maximizar
$\lfloor longitud/runPeriod\rfloor$ sobre runs da el mayor $k$.

\subsection{Complejidad}
La verificacion de igualdad por LCP es $\bigO(1)$ despues de preprocesar.
Enumerar runs puede hacerse en $\bigO(n)$ o $\bigO(n\log n)$ segun algoritmo.
Este es un problema dificil; para examen, asegura explicar periodo, LCP/LCS y
runs antes que intentar escribir implementacion completa.

\section{Imperial Roads}
\textbf{Link oficial.} \url{https://codeforces.com/gym/101889/problem/I}

\subsection{Enunciado oficial resumido}
Se tiene un grafo no dirigido, conectado y ponderado con $N$ ciudades y $R$
carreteras. Para cada consulta se da una carretera especifica que debe ser
reparada/incluida. Se pide el costo minimo de un conjunto de carreteras que
mantenga conectadas todas las ciudades e incluya esa carretera. Es decir: MST
forzando una arista.

\subsection{Solucion}
Construyo un MST $T$ con peso total $W$. Para una arista consultada
$e=(u,v,w)$, si la agrego a $T$, se crea un unico ciclo formado por $e$ mas el
camino entre $u$ y $v$ en $T$. Para volver a tener un arbol que incluya $e$,
debo quitar una arista de ese ciclo. Para minimizar el costo, quito la arista de
mayor peso en el camino $u-v$ del MST.
\[
ans=W+w-\operatorname{maxEdge}(u,v).
\]

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
build:
  T, W = Kruskal(graph)
  root T
  preprocess up[v][j] and mx[v][j]

maxEdge(u, v):
  ans = 0
  lift deeper node, updating ans
  lift both until LCA, updating ans
  include last edges to LCA
  return ans

query(edge u,v,w):
  return W + w - maxEdge(u,v)
\end{lstlisting}

\subsection{Correctitud}
Agregar $e$ al MST crea exactamente un ciclo. Cualquier arbol generador que
incluya $e$ y use las aristas de $T$ salvo una debe eliminar una arista del
camino $u-v$. El costo nuevo es $W+w-x$, donde $x$ es la arista eliminada. Para
minimizar, se maximiza $x$, por eso se quita la mayor arista del camino. Binary
lifting augmentado calcula ese maximo correctamente porque descompone el camino
en saltos de potencias de dos.

\subsection{Complejidad}
Kruskal $\bigO(R\log R)$. Preprocesamiento LCA $\bigO(N\log N)$. Cada consulta
$\bigO(\log N)$.

\section{Fools and Roads}
\textbf{Link oficial.} \url{https://codeforces.com/problemset/problem/191/C}

\subsection{Enunciado oficial resumido}
Se da un arbol de $n$ ciudades y $n-1$ carreteras. Luego se conocen varias rutas
simples usadas por personas, cada una entre dos ciudades. Para cada carretera se
debe contar cuantas rutas pasan por ella.

\subsection{Solucion}
Uso diferencia en arbol. Para una ruta $(u,v)$:
\[
cnt[u]++,\quad cnt[v]++,\quad cnt[\lca(u,v)]-=2.
\]
Luego hago DFS postorden. El acumulado que sube desde un hijo $x$ al padre es
la cantidad de rutas que usan la arista $(parent[x],x)$.

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
for each route (u,v):
  a = lca(u,v)
  cnt[u]++
  cnt[v]++
  cnt[a] -= 2

dfs(u,p):
  sum = cnt[u]
  for v in adj[u], v != p:
    child = dfs(v,u)
    ans[edge(u,v)] = child
    sum += child
  return sum
\end{lstlisting}

\subsection{Correctitud}
Los $+1$ en los extremos suben durante la acumulacion. Restar $2$ en el LCA
cancela ambas contribuciones antes de que pasen por encima del LCA. Por eso se
marcan exactamente las aristas del camino $u-v$. Como la suma es lineal, todas
las rutas pueden procesarse juntas.

\subsection{Complejidad}
Preprocesar LCA $\bigO(n\log n)$. Cada ruta $\bigO(\log n)$. DFS final
$\bigO(n)$.

\section{Frequency of String}
\textbf{Link oficial.} \url{https://codeforces.com/problemset/problem/963/D}

\subsection{Enunciado oficial resumido}
Se da un string $s$ y $n$ consultas. Cada consulta contiene un entero $k_i$ y un
patron $m_i$. Se pide la longitud minima de una substring de $s$ que contenga al
menos $k_i$ ocurrencias de $m_i$. Si $m_i$ aparece menos de $k_i$ veces, se
imprime $-1$. Los patrones son distintos y la suma de sus longitudes es hasta
$10^5$.

\subsection{Idea principal}
Si conocemos las posiciones de ocurrencia de $m$:
\[
p_1<p_2<\cdots<p_t,
\]
entonces la menor ventana que contiene $k$ ocurrencias consecutivas
$p_i,\ldots,p_{i+k-1}$ mide:
\[
p_{i+k-1}-p_i+|m|.
\]
La respuesta es el minimo de esa formula.

\subsection{Como obtener ocurrencias}
Como hay muchos patrones y texto fijo, Aho-Corasick es natural:
\begin{enumerate}[leftmargin=*]
\item Insertar todos los patrones en un trie.
\item Construir failure links.
\item Recorrer $s$ una vez.
\item Registrar en que posiciones termina cada patron.
\end{enumerate}
Tambien puede razonarse con suffix array, pero Aho-Corasick se ajusta mejor a
muchos patrones distintos.

\subsection{Pseudocodigo}
\begin{lstlisting}[language=C++]
answer(pattern id, k):
  occ = occurrences[id] // posiciones de inicio ordenadas
  if occ.size() < k: return -1
  ans = INF
  len = pattern_length[id]
  for i = 0; i+k-1 < occ.size(); i++:
    ans = min(ans, occ[i+k-1] - occ[i] + len)
  return ans
\end{lstlisting}

\subsection{Correctitud}
Cualquier substring que contiene $k$ ocurrencias tiene una primera y una
k-esima ocurrencia entre las posiciones ordenadas. Si esas son $p_i$ y
$p_{i+k-1}$, la substring debe cubrir desde $p_i$ hasta el final de la
ocurrencia en $p_{i+k-1}$, longitud al menos $p_{i+k-1}-p_i+|m|$. Esa ventana
existe y contiene esas $k$ ocurrencias, por lo que revisar ocurrencias
consecutivas da el optimo.

\subsection{Complejidad}
Aho-Corasick: $\bigO(|s|+\sum |m_i|+occ)$ segun implementacion de salida. Las
ventanas se calculan linealmente en el numero de ocurrencias de cada patron.
Detalle: las ocurrencias pueden solaparse y deben contarse.
